# Falsification Policy

Version: `v0.1`
Last updated: 2026-06-09

## Purpose

The project should try to disprove its own thesis.

Each hypothesis must have:

- What would support it.
- What would weaken it.
- What would contradict it.
- Current confidence.
- Evidence links.

## Weekly Review

During each weekly run:

1. Review new evidence.
2. Map evidence to hypotheses.
3. Mark direction: supports, weakens, contradicts, ambiguous.
4. Create an audit entry for weakens/contradicts.
5. Update hypothesis confidence only after review.

## Audit Output

Falsification audits should write to:

`audits/falsification/YYYY-MM-DD.md`

Do not silently edit old findings. Preserve prior thesis state and create a thesis update if the main view changes.
