# Trend Discovery Strategy

Version: `v0.1`
Last updated: 2026-06-09

## Purpose

The research agent must not be limited to initial examples like Microsoft Copilot or Doubao. Those are anchor cases, not the boundary of the project.

The agent should begin each daily or weekly run by asking:

> What changed in AI-to-C business models, and does it reveal a new research direction?

## Discovery Before Case Study

The agent should use this funnel:

1. Trend signal: a new market observation, launch, pricing change, user behavior shift, regulation, acquisition, shutdown, or monetization experiment.
2. Research idea: a question worth investigating across multiple products or categories.
3. Target model: a business-model pattern that may deserve deeper analysis.
4. Candidate product/company: a specific case that can test the idea.
5. Case study: a promoted target with enough evidence and strategic importance.

Not every signal deserves a case study.

## Signal Categories

Track signals in these categories:

- Pricing change: new paid tier, discount, annual plan, family plan, student plan, bundle, or price increase.
- Usage limit change: message caps, model throttling, credit packs, generation limits, file limits, or priority access.
- Monetization experiment: ads, affiliate, marketplace, revenue share, creator economy, data licensing, or commerce.
- Distribution shift: default placement in OS, browser, search, app store, device, telecom bundle, or super-app ecosystem.
- Retention signal: churn, engagement, habit formation, DAU/MAU, app ranking durability, or cohort evidence.
- Cost signal: model cost changes, inference optimization, heavy-user controls, or media generation economics.
- Product behavior shift: move from chatbot to agent, companion, tutor, creator tool, workflow automation, or personal OS.
- Company strategy: acquisition, shutdown, funding, hiring, partnership, enterprise pull-through, or ecosystem defense.
- Regional market signal: China, US, EU, India, Japan, Southeast Asia, or other geography-specific pattern.

## Promotion Rules

Promote a signal to a research idea when at least two are true:

- It affects monetization, retention, distribution, or cost.
- It appears in more than one company or category.
- It is a clear counterexample to the current thesis.
- It reveals a new target user segment.
- It changes the survival classification of a product.
- It may explain why large companies are advantaged.

Promote a research idea to a case-study candidate when:

- There is enough source evidence to build a profile.
- The product is representative of a broader pattern.
- The case could change the main answer.
- The case adds category or geography coverage not already represented.

## Daily Run Question

Each daily run should answer:

- What changed?
- Is it AI-to-C?
- Which business-model lever changed?
- Is it a one-off event or part of a pattern?
- Should it become a signal, research idea, or case-study candidate?

## Weekly Run Question

Each weekly run should answer:

- Which signals repeated?
- Which trends look stronger?
- Which products should be promoted to deeper research?
- Which existing hypotheses gained or lost confidence?
- What should the agent investigate next week?
