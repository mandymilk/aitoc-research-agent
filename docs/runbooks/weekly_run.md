# Weekly Runbook

## Objective

Turn daily trend signals into research priorities.

## Steps

1. Review all daily run notes from the week.
2. Cluster signals by business-model lever.
3. Identify repeated patterns across products or markets.
4. Promote research ideas into target models or case-study candidates.
5. Update the case-study backlog.
6. Decide whether any thesis confidence changed.
7. Create next week's research agenda.

## Output

- `trend_radar/runs/weekly/YYYY-MM-DD.md`
- Updated research ideas.
- Updated case-study backlog.
- Optional thesis update.

## Weekly Decision Questions

- Did any company move from free AI usage toward tiered monetization?
- Did any product add ads, commerce, marketplace, or bundle revenue?
- Did any category show evidence of stronger or weaker retention?
- Did any startup demonstrate standalone viability?
- Did any large platform show a new subsidy or distribution advantage?
- Did a new category emerge that should be added to the taxonomy?
